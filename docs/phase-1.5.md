# Phase 1.5 — 방향성 제시 (AI 연계 확장)

## 목표

Phase 1의 수동 입력 중심 흐름에 AI를 얹어, 기록 허들을 낮추고 내러티브 구축 기능을 추가한다.
Phase 1이 “기록 → 분석 → 출력”이었다면, Phase 1.5는 “대화로 기록 → 전략 수립”이다.

## 기능 명세

### 2. 커리어 아카이브 — AI 확장 (`app/(main)/archive/`)

#### AI 대화형 경험 추출

- 챗 인터페이스에서 자유롭게 경험을 이야기하면 AI가 구조화된 기록으로 변환
- 추출 결과를 유저가 확인·수정 후 저장
- 대화 흐름 예시:
  - AI: “최근에 가장 몰입했던 활동이 뭐였어?”
  - 유저: 자유 응답
  - AI: 역할, 기간, 성과, 키워드를 자동 태깅하여 제시

#### 파일 파싱

- PDF, 이미지(활동 확인서, 수료증 등) 업로드 → AI가 내용 추출
- 추출된 정보를 아카이브 입력 폼에 자동 채움
- 유저 확인 후 저장

### 4. 전략·설계 (`app/(main)/strategy/`)

#### 진로 로드맵

- 현재 기록 기반으로 “앞으로 쌓으면 좋을 경험” 추천
- 관심 직무와 현재 역량 갭 분석
- 타임라인 형태로 시각화

#### 내러티브 구축

- 흩어진 경험들을 하나의 스토리라인으로 연결
- “나는 ___한 사람이다”를 경험 근거와 함께 생성
- 자소서·면접에 바로 활용 가능한 서사 구조 제공

#### 가상 자소서 미리보기

- 특정 기업·직무를 선택하면 현재 기록 기반으로 자소서 초안 미리보기
- 부족한 부분 하이라이트 → 아카이브 추가 입력 유도

## Phase 1과의 관계

- Phase 1의 아카이브 입력 UI는 그대로 유지하되, AI 입력 채널을 병렬로 추가
- 전략(4) 기능은 분석(3) 데이터에 의존 → Phase 1의 분석이 완성된 후 구현
- AI API 호출은 모두 백엔드 엔드포인트 경유 (CLAUDE.md 규칙 준수)

## 우선순위

1. AI 대화형 경험 추출 — 서비스 차별점, 기록 허들 감소의 핵심
2. 내러티브 구축 — “왜 ARC를 쓰는가”에 대한 답
3. 파일 파싱 — 기존 자료 마이그레이션 편의
4. 진로 로드맵 — 분석 데이터 축적 후 의미 있음
5. 가상 자소서 미리보기 — 익스포트(5)와 연계, 후순위 구현 가능
