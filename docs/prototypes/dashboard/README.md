# ARC 대시보드 프로토타입 — 사용자 상태 분기 A / B / C

> Notion 「대시보드 포함 항목 및 디자인 기획」(ARC 대시보드 기능명세서 v1.0)을 바탕으로
> 만든 **정적 시안(프로토타입)**입니다. 실제 제품 코드가 아니며, 기획 검토용입니다.

## 무엇인가
- 누적 기록 수를 기준으로 한 3단계 상태(A/B/C)별 대시보드 화면 시안
- ARC 디자인 시스템(`app/globals.css`)의 토큰·타이포그래피를 그대로 이식해 **기존 UI를 따름**
- 차트(도넛/육각 레이더/막대/갭 레이더)는 외부 라이브러리 없이 SVG·CSS로 직접 그림 (현 제품과 동일한 방식)

## 파일
| 파일 | 설명 |
|---|---|
| `index.html` | **A/B/C 토글**이 달린 인터랙티브 뷰어 (가장 먼저 열어보세요) |
| `state-a.html` | 상태 A — 온보딩 시작 (기록 0개) |
| `state-b.html` | 상태 B — 초기 기록 (기록 1~3개) |
| `state-c.html` | 상태 C — 성장 / 풀 대시보드 (기록 4개 이상) |
| `arc-dashboard-state-a.png` | 상태 A 데스크톱 풀페이지 캡처 |
| `arc-dashboard-state-b.png` | 상태 B 데스크톱 풀페이지 캡처 |
| `arc-dashboard-state-c.png` | 상태 C 데스크톱 풀페이지 캡처 |
| `arc-dashboard-state-c-mobile.png` | 상태 C 모바일(375px) 캡처 — 1열 반응형 |
| `tokens.css` / `app.css` | 디자인 토큰 / 컴포넌트 스타일 |

## 보는 법
```bash
cd docs/prototypes/dashboard
python3 -m http.server 8137
# 브라우저에서 http://127.0.0.1:8137/  (또는 index.html)
```
> ⚠️ HTML을 `file://`로 직접 열면 일부 브라우저에서 CSS가 안 붙을 수 있어요. 위처럼 로컬 서버로 여세요.

## 상태별 핵심 차이 (명세서 §7 기준)
- **A**: 빈 화면 미노출 → 큰 CTA + 온보딩 가이드 + 샘플 미리보기 + 블로그. (지표·차트·설계·채용 전부 숨김, 광고/채용 노출 금지)
- **B**: 3 KPI 카드 + 도넛(부분) + 막대·레이더 **🔒잠금** + 갭레이더(준비중) + 오늘의 한 가지 + **분석 진척 N/4** + AI 챗봇(크게) + 추천활동·블로그
- **C**: 풀 대시보드 — 도넛·육각 레이더·막대·갭레이더 + 주간 다이제스트 + 강점 재발견 + 행동/설계/확장(채용 조건부) 전체

## 기획안 vs 현재 운영 화면 (검토 포인트)
- KPI 위젯을 기획안대로 **3카드(누적 경험·강점·키워드)**로 표현. 현 운영 대시보드는 4카드.
- 육각 레이더·강점 재발견·주간 다이제스트는 명세서상 **Phase 1.5** 항목 (시안엔 포함).
- 경험 유형 18종 컬러맵 미정 → 임시 팔레트 사용. 모든 수치는 더미.
