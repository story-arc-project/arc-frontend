## 1. 프로젝트 개요
---
### 카테고리
> 생산성 / 업무

### 한 줄 정의
AI 기반 포트폴리오 아카이빙 및 스토리라인 구축 서비스

### 배경
많은 사람들이 본인의 경험과 역량을 효과적으로 기록하고 정리하는 데 어려움을 겪고 있으며, 기존 포트폴리오 툴은 체계적인 입력 가이드라인이 부족합니다. 이에 AI 기술을 활용하여 개인의 경험을 심층 분석하고, 매력적인 스토리라인으로 재구성하여 자기 홍보와 직무 탐색에 실질적인 도움을 줄 수 있는 서비스의 필요성이 커지고 있습니다.

### 사용자 문제
사용자들은 자신의 경험과 역량을 체계적으로 정리하지 못하고 있으며, 자신을 효과적으로 홍보하는 데 어려움을 느낍니다. 또한 기존 기록 웹사이트나 툴은 제대로 된 입력 틀을 제공하지 않아 스스로 생각하고 기록해야 하는 부담이 큽니다.

### 해결 방식
AI 기반의 체계적인 경험 아카이빙 시스템을 통해 사용자의 스펙, 활동, 스토리, 역량 등을 손쉽게 기록하고 축적할 수 있도록 지원합니다. AI가 입력된 데이터를 분석하여 각 경험 간의 스토리라인을 구축하고 개인의 강점을 도출하며, 맞춤형 직무를 추천하여 효과적인 자기 브랜딩과 커리어 탐색을 돕습니다.

### 차별점
기록을 어려워하는 사용자들을 위해 AI와의 인터뷰 문답 형태의 입력 방식을 제공하며, 각 이력/경험/활동/역량별로 상세히 기술할 수 있는 가이드라인과 자유 템플릿을 제공하여 사용자 친화적인 입력 환경을 구축합니다. 단순한 기록을 넘어 AI가 개인의 고유한 서사를 발굴하고 미래 커리어 잠재력을 강조하는 스토리라인을 자동으로 생성한다는 점에서 차별점을 가집니다.

### 타겟 사용자
최근 졸업생 및 신입 구직자: 경력 초기 단계에서 자신의 경험을 체계적으로 정리하고, 효과적인 자기소개 및 직무 매칭을 원하는 사용자. 역량 개발에 힘쓰는 전문가: 자신의 역량과 경험을 지속적으로 업데이트하고 관리하며, 커리어 성장을 위한 개인화된 인사이트와 새로운 직무 기회를 탐색하는 사용자.

### 사용 시나리오
사용자는 서비스 초기 설정 시 자신의 기본적인 이력과 경험을 입력하거나 기존 파일을 가져와 포트폴리오를 구성합니다. 이후 새로운 활동이나 성과가 생길 때마다 '경험 아카이브' 페이지에서 구조화된 템플릿(미리 제공 질문 + 사용자가 추가한 블록)으로 기록을 업데이트합니다.

기록이 어렵거나 작성 중 막히는 순간에는, 사용자가 원할 때만 AI 인터뷰를 켜서 질문 도움을 받고 답변을 템플릿 필드로 채워 넣어 기록을 보완합니다(자동 전환 없음).

일상적인 작업의 시작점은 '대시보드'입니다. 대시보드는 입력 현황(완료율/미완료/최근 수정), 최근 항목(이어쓰기/빠른 편집), AI 분석 요약(특히 중요한 인사이트 하이라이트), 그리고 주요 페이지 바로가기(경험 아카이브·AI 분석·내보내기 등)로 구성된 ‘내 워크스페이스’로서, 사용자가 다음 행동으로 자연스럽게 이어지도록 돕습니다. 대시보드 내 ‘개인화된 직무 추천’은 요약 카드로만 제공하고, 상세 추천/탐색/수락 흐름은 'AI 분석' 페이지에서 수행합니다.

커리어 탐색 시 사용자는 'AI 분석' 페이지에서 강점과 스토리라인을 확인하고, 개인화된 직무 추천을 상세 근거와 함께 탐색/저장/수락할 수 있습니다. 완성된 포트폴리오는 '엑스포트' 기능을 통해 다양한 채용 플랫폼에 공유합니다.

### 제품 목표
사용자의 스펙, 활동, 이야기, 역량 등을 체계적으로 기록 및 축적할 수 있도록 돕고, AI 분석을 통해 개인의 강점과 잠재력을 발굴하여 맞춤형 커리어 가이드를 제공함으로써 효과적인 자기 브랜딩과 성공적인 직무 매칭을 지원합니다.

### 핵심 KPI
월간 활성 사용자(MAU) 및 재방문율, 사용자별 평균 기록 업데이트 빈도, AI 분석 보고서 생성 및 조회 수, AI 기반 직무 추천 수락률, 외부 채용 플랫폼 공유 기능 활용률, 사용자 만족도(피드백 점수).

### 리스크/이슈
AI 분석의 정확성 및 편향성 문제 발생 가능성, 사용자 데이터 보안 및 개인 정보 보호 문제, 사용자들의 적극적인 데이터 입력 유도 실패, 유사 경쟁 서비스 등장 및 차별성 약화, AI 모델 학습을 위한 충분한 데이터 확보의 어려움.

## 1. 사용자 경험 아카이빙 기능
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전

사용자가 자신의 이력, 경험, 활동, 역량 등을 체계적으로 기록하고 관리할 수 있는 UI/UX 환경을 제공합니다.

입력의 기본 경로는 ‘템플릿 기반 상세 입력’이며, 사용자는 서비스가 미리 제시한 템플릿 질문과 본인이 추가한 질문/블록만으로도 충분히 경험의 이야기를 완결해 기록할 수 있어야 합니다.

AI 인터뷰는 자동으로 강제 전환되는 단계가 아니라, 기록이 어려운 사용자(또는 작성 중 막히는 순간)를 돕기 위한 ‘선택적 도움 기능’입니다. 사용자는 언제든 AI 인터뷰를 켜거나 끌 수 있어야 하며, AI가 생성/정리한 내용은 사용자가 확정하기 전까지 ‘제안’으로 유지됩니다.

상세 입력 화면의 선택 경로는 아래 3가지로 명확히 제공합니다.
- A. 템플릿만으로 작성 완료(기본/권장)
- B. 작성 중 ‘도움받기’로 AI 인터뷰를 켜서 일부만 보완(부분 사용)
- C. 처음부터 AI 인터뷰로 시작(기록이 어려운 사용자용)

AI 인터뷰를 통해 얻은 답변은 템플릿 필드(예: 내 역할, 내가 한 일, 핵심 성과, 증빙 등)에 자연스럽게 채워 넣을 수 있도록 매핑을 제공하고, 사용자는 저장 전에 최종 내용을 검토/편집할 수 있어야 합니다.

**수용 기준:**
```
1. ○ 사용자는 다양한 형태(수동 텍스트 입력, 구조화된 양식 입력, 파일 업로드, 외부 서비스 동기화 등)로 경험 데이터를 입력할 수 있어야 한다.
2. ○ 각 경험 항목(이력, 활동, 역량 등)에 대해 상세한 가이드라인과 입력 템플릿이 제공되어 사용자가 쉽게 기록할 수 있어야 한다.
3. ○ 사용자는 미리 제공된 템플릿 질문과 본인이 추가한 질문/블록만으로도 경험 기록을 완결할 수 있어야 한다.
4. ○ AI 인터뷰는 기록이 어려운 사용자를 위한 선택적 도움 기능이어야 하며, 자동으로 강제 전환되지 않아야 한다.
5. ○ 사용자는 작성 중 언제든 AI 인터뷰를 켜거나 끌 수 있어야 하며, AI가 생성/정리한 내용은 사용자 확정 전까지 제안 상태로 유지되어야 한다.
6. ○ AI 인터뷰 답변을 템플릿 필드(예: 내 역할, 내가 한 일, 핵심 성과, 증빙 등)로 채워 넣을 수 있도록 매핑이 제공되어야 한다.
7. ○ 입력된 데이터는 시계열 또는 카테고리별로 분류하여 조회 및 편집할 수 있어야 한다.
```


### 1.1 라이브러리(경험 목록)
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web

사용자가 기록한 경험을 카드 형태로 모아보고, ‘라이브러리(저장된 필터)’ 단위로 묶어 탐색·수정·추가할 수 있는 목록 화면을 제공합니다.

- 카드 기본 정보: 경험명, 유형, 기간, 한 줄 요약, 상태(작성 중/완료), 마지막 수정일
- 기본 라이브러리: ‘전체’(시스템 기본)
- 사용자 라이브러리: 사용자가 직접 만든 저장된 필터(태그/상태/기간/유형 등 조건 조합)
  - 생성/편집/삭제(본인 것만)
  - 메타(선택): 라이브러리 이름, 색/아이콘
- 탐색: 검색, 정렬(최근 수정/기간/완료도), 필터(유형/상태/태그)
  - 현재 필터 상태를 ‘라이브러리로 저장’ 가능
  - 라이브러리 선택 시 카드 목록이 해당 조건으로 즉시 필터링
- 액션: 새 경험 추가, 카드 클릭으로 상세로 이동, 카드 퀵액션(편집/복제/보관)

태그는 경험 카드/상세 입력에서 부여되며, 라이브러리는 태그/필터를 활용해 경험을 폴더처럼 묶는 방식입니다.

AI 인터뷰는 이 화면에서 자동으로 시작되지 않으며, 사용자가 ‘도움받기’로 진입을 선택한 경우에만 상세 입력 화면에서 사용할 수 있습니다.


### 1.2 AI 인터뷰 도움받기(선택)
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web

기록이 어렵거나 작성 중 막히는 순간에만 사용자가 선택적으로 켤 수 있는 AI 인터뷰 모드를 제공합니다.

- 진입: 경험 상세 입력 화면의 ‘도움받기’ 버튼(또는 ‘AI로 시작하기’)으로만 진입
- 종료: 언제든 끄기(템플릿 화면으로 복귀)
- 동작: AI가 질문을 제안하고 사용자가 답하면, 답변을 관련 템플릿 필드(예: 내 역할/내가 한 일/핵심 성과/증빙)로 매핑해 채워 넣음
- 안전장치: AI가 채운 내용은 사용자가 확정하기 전까지 ‘제안’ 상태이며, 사용자 편집/되돌리기 가능

‘상세 입력 완료 후 자동 인터뷰 전환’은 허용하지 않습니다.


### 1.3 경험 상세 입력(템플릿 기반)
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web

경험 기록의 기본 경로로, 미리 제공된 템플릿 질문과 사용자가 추가한 질문/블록으로 경험을 끝까지 작성할 수 있는 상세 입력 화면을 제공합니다.

- 공통 코어 입력: 경험명, 기간, 유형, 한 줄 요약, 내 역할/기여도, 핵심 성과, 증빙(파일/링크)
- 확장 입력(선택): 배경/목표/행동/성과/배운 점, 사용한 스킬, 협업/팀, 난이도/임팩트, 공개 설정
- 사용자 확장: 질문/블록 추가·삭제·순서 변경, 섹션 접기/펼치기
- 저장: 초안 저장(작성 중), 완료 처리(완료)

사용자 추가 템플릿 프리셋(블록 기반)
- 목적: 사용자가 ‘반복 셀(로그)’ 중심의 자주 쓰는 입력 양식을 블록으로 조합해, 다른 경험에도 재사용할 수 있도록 함
- 블록(부품) 유형(예)
  - 텍스트 박스(짧은/긴)
  - 체크리스트(여러 개 선택) / 단일 선택(하나 선택)
  - 날짜/기간
  - 태그(역량/기술/키워드)
  - 링크(URL + 설명)
  - 파일 첨부(파일 + 설명 + 증빙 유형)
  - 반복 셀(로그형: 행 추가로 동일 구조를 누적)
  - 표(고정 열 + 행 추가)
- 프리셋 저장: 사용자는 현재 경험에서 추가한 블록/반복 셀을 선택해 ‘프리셋으로 저장’할 수 있으며, 이름/설명(선택)/추천 경험 유형(선택)을 관리한다.
- 프리셋 적용
  - 기본: 현재 경험의 확장 입력 영역에 ‘삽입(추가)’
  - 선택: 현재 경험의 확장 입력 영역을 ‘덮어쓰기(교체)’
  - 공통 코어 입력은 덮어쓰기 대상이 아니며, 사용자 확인 및 되돌리기를 제공한다.

AI 인터뷰는 ‘도움받기’로 사용자가 켰을 때만 활성화되며, 템플릿 작성 흐름을 방해하지 않습니다.

#### 1.3.20 템플릿 프리셋 저장·관리

사용자는 자신이 구성한 블록 조합(특히 반복 셀 중심의 로그/성과/작업 기록 양식)을 ‘템플릿 프리셋’으로 저장하고 재사용할 수 있어야 한다.

프리셋 저장
- 사용자는 현재 경험의 확장 입력 영역에서 블록(1개 이상)을 선택해 ‘프리셋으로 저장’할 수 있어야 한다.
- 저장 시 입력: 프리셋 이름(필수), 설명(선택), 추천 경험 유형(선택)

프리셋 관리
- 사용자는 내 프리셋 목록을 조회할 수 있어야 한다.
- 사용자는 내 프리셋을 검색/즐겨찾기(선택)/복제/이름 변경/삭제할 수 있어야 한다.
- 프리셋 수정은 기존 프리셋을 덮어쓰거나(간단 수정), ‘복제 후 수정’ 흐름을 제공할 수 있어야 한다.

안전장치
- 프리셋은 공통 코어 입력(경험명/기간/유형/요약/역할/성과/증빙)을 직접 변경하지 않는다.
- 프리셋은 확장 입력 영역의 블록 구성만 대상으로 한다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web


#### 1.3.21 템플릿 프리셋 적용(삽입/덮어쓰기)

사용자는 저장된 템플릿 프리셋을 현재 경험의 확장 입력 영역에 적용할 수 있어야 한다.

적용 방식
- 삽입(추가): 기존 확장 입력 블록을 유지한 채, 프리셋 블록 세트를 지정 위치에 추가한다.
- 덮어쓰기(교체): 기존 확장 입력 블록을 프리셋 블록 세트로 교체한다.

적용 범위 정책
- 덮어쓰기는 ‘확장 입력 영역’에 한해 허용한다.
- 공통 코어 입력은 항상 유지되며, 프리셋으로 덮어쓸 수 없다.

사용자 확인 및 되돌리기
- 덮어쓰기 적용 전, 교체될 블록 범위를 사용자에게 명확히 안내해야 한다.
- 적용 직후 ‘되돌리기’를 제공해야 한다.

권장 UX
- 프리셋 선택 시 미리보기(블록 구성/반복 셀 열/예시 행)를 제공한다.
- 자주 쓰는 프리셋은 즐겨찾기하거나 상단에 고정할 수 있다(선택).
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web

> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: User | **기기**: Web

경험 유형(예: 전공 및 수강 수업/성적, 대외활동, 인턴 및 업무 경력 등)에 따라 기본 템플릿 질문과 ‘유형별 확장 입력 필드(프리셋)’ 구성이 달라지도록 템플릿 프리셋을 제공합니다.

- 사용자는 경험 생성 시 유형을 선택하고, 해당 유형의 기본 템플릿을 불러옴

- 유형별 섹션(확장)의 기준은 [기능 '경험 상세 입력(템플릿 기반)'] 하위의 ‘유형별 템플릿(확장) 스펙’으로 정의하며, 유형 선택 시 해당 스펙을 기반으로 필드를 로드
- 사용자는 유형별 템플릿을 기반으로 질문/블록을 자유롭게 추가·수정 가능

AI 인터뷰는 템플릿을 대체하지 않으며, 필요 시에만 보조 수단으로 사용합니다.

## 2. AI 기반 포트폴리오 분석 및 스토리라인 구축

```

4. ○ 사용자는 AI가 생성한 스토리라인에 대해 직접 편집하거나 피드백을 제공하고, AI가 이를 반영하여 수정할 수 있도록 요청할 수 있어야 한다.
```
